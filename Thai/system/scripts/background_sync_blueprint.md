# พิมพ์เขียว: ระบบเดมอนซิงค์เบื้องหลัง (`sync_daemon.py`)

## 1. เป้าหมาย
ผลักภาระงานแปลไฟล์และการรวบรวมบริบทหลักด้วยมือไปให้สคริปต์ Python เบื้องหลัง (Background Worker) จัดการแทน เพื่อประหยัดโควต้าโทเคนและเวลาในการประมวลผลสำหรับเอเจนต์หลัก

## 2. สิ่งที่ต้องใช้ (Dependencies)
- `watchdog`: สำหรับตรวจสอบการเปลี่ยนแปลงของระบบไฟล์
- `openai` / `anthropic` / `google-genai`: สำหรับเรียกใช้ LLM API ในการแปลภาษา
- `dotenv`: สำหรับโหลดคีย์ API

## 3. สถาปัตยกรรมและขั้นตอนการทำงาน

### A. ตัวเฝ้าระวัง (The Watcher)
- เปิดใช้งาน `watchdog.Observer` เพื่อเฝ้าดูแฟ้ม `English/`
- กรองอีเวนต์: รับฟังเฉพาะอีเวนต์ `on_created` (สร้างใหม่) และ `on_modified` (แก้ไข) ของไฟล์ `.md`
- เพิกเฉยอีเวนต์ที่เกิดจากไฟล์ `English/system/session_log.md` เพื่อป้องกันการทำงานวนลูปไม่รู้จบระหว่างการบันทึกประวัติการทำงาน

### B. คิวพักคำสั่ง (The Debouncer)
- การเปลี่ยนแปลงไฟล์อาจเกิดขึ้นรัวๆ (เช่น เมื่อสคริปต์เขียนข้อมูลหลายส่วนติดกัน)
- ใช้ระบบจับเวลาหน่วง (Debounce Queue) เช่น รอ 5 วินาทีหลังจากการแก้ไขครั้งล่าสุดก่อนจะสั่งซิงค์

### C. พนักงานซิงค์ (The Sync Worker)
เมื่อตัวจับเวลาหน่วงทำงานสำหรับไฟล์ `English/path/to/file.md`:
1. **การซิงค์แบบแปลภาษา:**
   - อ่านเนื้อหาภาษาอังกฤษ
   - ส่งไปยัง LLM API ด้วยคำสั่ง: *"แปลไฟล์ markdown นี้เป็นภาษาไทยอย่างละเอียด โดยคงรูปแบบเดิมไว้ทั้งหมด"*
   - บันทึกผลลัพธ์ลงใน `Thai/path/to/file.md` (สร้างโฟลเดอร์อัตโนมัติหากยังไม่มี)
2. **การรวบรวมบริบท (Context Compilation):**
   - รันตรรกะการต่อไฟล์ (วนซ้ำอ่านไฟล์ `English/**/*.md` ทั้งหมด)
   - เขียนทับไฟล์ `Coprem_All_Data_Context.md` ที่หน้าแรก (Root)

## 4. การจัดการข้อผิดพลาด (Error Handling)
- หาก API แปลภาษาล้มเหลว ให้บันทึกข้อผิดพลาดลงใน `system/scripts/logs/sync_error.log` และลองใหม่ด้วยระบบทวีคูณเวลา (Exponential Backoff)
- ระบบเดมอนต้องทำงานตลอดเวลา (เช่น ผ่าน `pm2` หรือบริการ `systemd`)
